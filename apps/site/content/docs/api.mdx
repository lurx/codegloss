# Component API

CodeGloss is a Web Component (`<code-gloss>`) with a thin React wrapper at `codegloss/react`. The configuration shape is the same in both — only the way you pass it differs.

## Web Component

````html
<script type="module" src="https://unpkg.com/codegloss/dist/index.js"></script>

<code-gloss>
  <script type="application/json">
    {
      "code": "...",
      "lang": "js",
      "filename": "example.js",
      "annotations": [...],
      "connections": [...]
    }
  </script>
</code-gloss>
````

The element reads its config from a `<script type="application/json">` child. JSON keeps complex props (arrays, objects) clean — no attribute serialization headaches.

## React wrapper

````js
import { CodeGloss } from 'codegloss/react';

<CodeGloss
  code="..."
  lang="js"
  filename="example.js"
  annotations={[...]}
  connections={[...]}
/>
````

The React wrapper is a thin JSX adapter. Under the hood it renders the same `<code-gloss>` custom element with a JSON config child.

## Configuration

The same `CodeGlossConfig` shape powers both usage modes:

| Field | Type | Default | Description |
| --- | --- | --- | --- |
| `code` | `string` | — | Raw source code to display. Preserve indentation. |
| `lang` | `string` | — | Language identifier (`js`, `ts`, `py`, `css`, etc.). |
| `filename` | `string?` | — | Filename shown in the toolbar. |
| `annotations` | `Annotation[]?` | `[]` | Token-level annotations with inline callouts. |
| `connections` | `Connection[]?` | `[]` | Visual arcs between related annotations. |
| `runnable` | `boolean?` | `lang === 'js'` | Whether to show the Run button. |
| `theme` | `string?` | — | Named theme (`github-dark`, `dracula`, etc.). |

## Types

### Annotation

Defines a single highlighted token with an expanding callout.

````ts
type Annotation = {
  id: string;          // Unique ID, e.g. "a1"
  token: string;       // Exact substring to match, e.g. "memo[k]"
  line: number;        // 0-indexed line number
  occurrence: number;  // 0-indexed match on that line
  title: string;       // Callout heading
  text: string;        // Callout body
};
````

- **`token`** — the exact substring to search for on the given line. Must match character-for-character.
- **`line`** — 0-indexed. Line 0 is the first line of the code.
- **`occurrence`** — when a token appears multiple times on one line, this selects which instance (0 = first).

Clicking an annotated token expands an inline callout row directly below the line, with smooth animation. Click again or press Escape to dismiss.

### Connection

Draws a visual arc between two annotations in the line number gutter.

````ts
type Connection = {
  from: string;   // Source annotation ID
  to: string;     // Target annotation ID
  color: string;  // Hex color, e.g. "#534AB7"
  title?: string; // Popover heading shown on click
  text?: string;  // Popover body. Without it, the arc is not clickable.
};
````

When a connection has `text`, the arc and its endpoint dots become clickable. Clicking opens a popover (using the native [Popover API](https://developer.chrome.com/blog/introducing-popover-api)) with the title and text — useful for explaining *why* two pieces of code are related.

Arcs render as dashed Bezier curves with dots at each endpoint. Multiple connections are offset horizontally to avoid overlap.

## Custom syntax highlighting

The Web Component uses a built-in regex highlighter by default (recognizes JS/TS keywords, strings, numbers, comments). To plug in Shiki, Prism, or any HTML-emitting highlighter, set the `highlight` property on the element directly:

````js
import { codeToTokens } from 'shiki';

const el = document.querySelector('code-gloss');

el.highlight = (code, lang) => {
  const { tokens } = codeToTokens(code, { lang, theme: 'github-dark' });
  return tokens.map((line) =>
    line.map((token) =>
      `<span style="color:${token.color}">${escapeHtml(token.content)}</span>`
    ).join('')
  );
};
````

The function receives the full code and language, and returns an array of HTML strings — one per line. Annotation marks are injected into the highlighted HTML automatically using position mapping through the tag structure.

> Because functions can't be serialized to JSON, `highlight` is set as a property on the element, not via the JSON config child. The React wrapper exposes it through a ref / property setter.

## Theming

### Named themes

Apply a bundled theme via the `theme` attribute (HTML) or prop (React/Vue/Svelte):

````html
<code-gloss theme="github-dark">
````

````jsx
<CodeGloss code="..." lang="js" theme="dracula" />
````

**Bundled themes:**

| Theme | Variant | Based on |
| --- | --- | --- |
| `github-light` | light | GitHub code blocks |
| `github-dark` | dark | GitHub code blocks |
| `one-light` | light | Atom One Light |
| `one-dark` | dark | Atom One Dark |
| `dracula` | dark | Dracula |
| `nord-light` | light | Nord Snow Storm |
| `nord-dark` | dark | Nord Polar Night |
| `vitesse-light` | light | Vitesse by antfu |
| `vitesse-dark` | dark | Vitesse by antfu |

Each theme sets both **syntax token colors** (keywords, strings, numbers, comments) and **chrome colors** (background, borders, toolbar, annotations). Dark-only themes (like Dracula) apply unconditionally. Themes with a single variant ignore `prefers-color-scheme`.

### Global theme via remark plugin

Set a default theme for all code blocks:

````js
remarkCodegloss({ theme: 'github-dark' })
````

Individual blocks can override the global default by specifying their own `theme`.

### Global theme in vanilla JS

````js
import { applyGlobalTheme } from 'codegloss';

// Applies 'github-dark' to all <code-gloss> elements on the page
// that don't already have a theme attribute.
applyGlobalTheme('github-dark');
````

### Theme resolution

The `theme` is resolved in this order (first one wins):

1. `theme` HTML attribute on the element
2. `theme` in the JSON config
3. Global theme set via `applyGlobalTheme()` or the remark plugin's `theme` option

### Programmatic theme access

````ts
import { resolveTheme } from 'codegloss';
import type { CodeGlossTheme, CodeGlossThemeVariant } from 'codegloss';

const theme = resolveTheme('dracula');
// theme.name === 'dracula'
// theme.dark contains all color values
````

Themes are also available as individual tree-shakeable imports:

````ts
import { dracula, githubLight, oneDark } from 'codegloss/themes';
````

### CSS variable overrides

For fine-grained control, override individual CSS custom properties on the `code-gloss` element. **CSS variable overrides always win over theme values** because they're more specific than `:host` rules:

````css
code-gloss {
  --cg-bg: #1a1a1a;
  --cg-border: #333;
  --cg-text: #e0e0e0;
  --cg-ann-border: #ff6b9d;
}
````

| Property | Light default | Dark default | Description |
| --- | --- | --- | --- |
| `--cg-bg` | `#f8f8f6` | `#1e1e1e` | Code area background |
| `--cg-border` | `#e8e8e6` | `#3a3a3a` | Border color |
| `--cg-text` | `#1a1a1a` | `#e0e0e0` | Code text color |
| `--cg-muted` | `#666` | `#999` | Secondary text |
| `--cg-line-num` | `#bbb` | `#555` | Line number color |
| `--cg-toolbar-bg` | `#f8f8f6` | `#2a2a2a` | Toolbar background |
| `--cg-badge-bg` | `#efefed` | `#3a3a3a` | Lang badge background |
| `--cg-ann-bg` | `rgba(83,74,183,0.12)` | `rgba(174,169,236,0.12)` | Annotation highlight |
| `--cg-ann-border` | `#534ab7` | `#afa9ec` | Annotation underline |
| `--cg-ann-hover` | `rgba(83,74,183,0.22)` | `rgba(174,169,236,0.22)` | Annotation hover state |

Dark mode activates automatically via `prefers-color-scheme: dark`. To force a theme regardless of OS preference (e.g. tied to a site theme toggle), use a named theme or override the variables on the host element directly.

### External highlighter interaction

When using a custom `highlight` function (Shiki, Prism, etc.), the **syntax token colors** from the theme are unused — the external tool owns the token markup. Only the **chrome** portion of the theme applies (background, borders, annotations, toolbar).

## No CSS import required

You don't need to import a stylesheet anywhere. The Web Component uses Shadow DOM with constructable stylesheets — all styles are bundled into the JS chunk and applied automatically when the element registers.
